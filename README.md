# HyCAN-DB: Hydrogen in Carbon Nanomaterials Database

[![License: MIT](https://img.shields.io/badge/Code-MIT-blue.svg)](LICENSE)
[![Data License: CC-BY 4.0](https://img.shields.io/badge/Data-CC--BY%204.0-lightgrey.svg)](https://creativecommons.org/licenses/by/4.0/)

HyCAN-DB is an open, FAIR-compliant database of hydrogen sorption measurements
in carbon nanomaterials (activated carbons, carbon nanotubes, graphene, MOF-derived
carbons, and related porous frameworks). It aggregates primary data extracted from
peer-reviewed literature, applies a rigorous curation and harmonisation pipeline,
and provides ready-to-use benchmark datasets for machine-learning models that predict
gravimetric or volumetric hydrogen uptake from structural and synthesis descriptors.
The project is designed to accelerate reproducible research at the intersection of
hydrogen energy storage and data-driven materials science.

---

## Why this project exists

Published hydrogen-storage measurements are scattered across hundreds of journal
articles in incompatible units, reported under inconsistent conditions, and rarely
shared as machine-readable files. HyCAN-DB solves this by (1) systematically
extracting and digitising isotherms from the literature, (2) converting all data to
consistent SI units with full provenance metadata, (3) flagging outliers and
transcription errors with an auditable quality-control log, and (4) releasing
everything under open licences so the community can build on it immediately.

## Repository structure

```
hycan-db/
├── data/
│   ├── raw/            # Unmodified source files (Excel, CSV) — not tracked by git
│   ├── interim/        # Partially processed intermediate files
│   ├── processed/      # Final, analysis-ready datasets
│   └── external/       # Third-party reference data (e.g. NIST, DoE targets)
├── docs/
│   └── data_dictionary.md   # Field definitions and controlled vocabularies
├── figures/            # Publication-quality figures (generated by scripts/notebooks)
├── manuscripts/
│   ├── data_descriptor/     # Journal Data Descriptor draft
│   └── poster/              # Conference poster source files
├── notebooks/          # Jupyter notebooks (numbered, exploratory)
├── references/
│   └── papers/         # Source PDFs — not tracked by git
├── scripts/            # Standalone processing and analysis scripts
├── src/
│   └── hycan/          # Python package: ingestion, cleaning, validation, ML utils
├── tests/              # pytest test suite
├── CHANGELOG.md
├── CITATION.cff
├── pyproject.toml
└── requirements.txt
```

## Installation

Requires Python 3.10+.

```bash
# 1. Clone the repository
git clone https://github.com/[username]/hycan-db.git
cd hycan-db

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows

# 3. Install dependencies
pip install --upgrade pip
pip install -r requirements.txt

# 4. Install the hycan package in editable mode
pip install -e .
```
